Create / Import XRPL Account

Import an XRPL Account

Import an account using either your family seed (secret) or a hex private key. The page uses the xrpl library to derive your XRPL address and public key. Use only for learning; never paste a key that holds real funds in a browser.

Signing algorithms: The XRP Ledger supports both secp256k1 and Ed25519 for key pairs. In these tutorials we use secp256k1 (the same curve used by Bitcoin and by the Blockchain Concepts key-generation section). If you use a different tool or library, ensure the algorithm matches—the same seed can produce a different address under Ed25519 than under secp256k1.

From Seed to Private Key

The family seed (secret) is a compact value used to derive the account’s private and public keys. The diagram below shows how the seed is used to create the private key (and then the public key and address) for Ed25519.

Ed25519: The seed is hashed (e.g. SHA-512Half) to produce the private key; the public key is then derived from the private key. The address is derived from the public key.

This flow is for Ed25519. For secp256k1 (used in these tutorials), the general flow is the same—seed → private key → public key → address—but the derivation is more involved: the XRP Ledger’s secp256k1 process derives a root key pair from the seed, then an intermediate key pair, and combines them to get the master key pair. Not all 32-byte values are valid secp256k1 private keys, so the algorithm may need to retry with a different sequence. The XRPL key derivation docs describe both algorithms in detail.

X-Address vs classic address: The classic address (e.g. r...) is the original XRPL format. An X-Address encodes the same account plus an optional destination tag and the network (mainnet or testnet) in a single string (e.g. X...). That way you can send to one value instead of address + tag separately.

Destination tag: A destination tag is a number (0–232−1) that some exchanges or services require when you send them XRP. They use one XRPL address for many users and rely on the tag to credit the right account. If a service asks for a destination tag, you can use an X-Address that includes that tag, or send the classic address and tag as two fields.

Understanding the Process

  1. Sign Transaction: Create and sign a transaction using your family seed or private key (see Sign Transaction page).
  2. Submit to Network: Send the signed tx_blob to an XRPL node via the submit command (WebSocket/JSON-RPC).
  3. Propagation: The node broadcasts your transaction to other servers in the network.
  4. Consensus: Validators run the XRP Ledger Consensus Protocol to agree on the set and order of transactions.
  5. Ledger Close: A new ledger version is closed (typically every 3–5 seconds) including your transaction if it is valid.
  6. Validation: Once in a validated ledger, the transaction is final.

Checking Transaction Status

After submitting, you can check your transaction status on:

  • XRPL.org Explorer: https://livenet.xrpl.org
  • Bithomp (mainnet): https://bithomp.com/explorer

On the explorer, you can see:

  • Transaction status: Success or failure (e.g. tec*, tesSUCCESS)
  • Ledger index: Which validated ledger contains the transaction
  • Fee: XRP destroyed (burned) as the transaction fee
  • Account / Destination: Sender and recipient
  • Amount: Value transferred
Important Notes:
  • Ensure the account has enough XRP to cover the payment amount plus the fee (and reserve if the destination is new).
  • Transactions on the XRP Ledger are final once included in a validated ledger; there is no reversal.
If Your Transaction Is Not Accepted:
  • Check Sequence: Use the account's current sequence (from account_info) for the next transaction. If the transaction fails with a sequence error, ensure you're not reusing or skipping sequence numbers.
  • Check Balance: Verify the account has sufficient XRP for the amount, fee, and reserve (if creating a new account or trust line).
  • Check LastLedgerSequence: If the transaction has a LastLedgerSequence, it will expire if not included before that ledger. Submit again with a new LastLedgerSequence if needed.
  • Check engine_result: The server returns an engine result (e.g. tecUNFUNDED, tecNO_PERMISSION). Use the error message to correct the transaction and try again.

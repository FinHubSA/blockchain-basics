Signing XRPL Transactions

This page demonstrates how to create and sign a Payment transaction on the XRP Ledger. Fill in the transaction details below, then sign with your family seed or private key. If you created or imported an account on the Create / Import Account page, you can use the saved credentials. Use only for learning; never sign transactions that move real funds in a browser.

Note: This is a simplified Payment flow. The XRP Ledger supports other transaction types (OfferCreate, TrustSet, Escrow, etc.); those require additional fields and can be built manually or with the xrpl library.

Transaction Details

Understanding the transaction components:

  • Account: The sender’s classic address. The account’s key pair must sign the transaction.
  • Destination: The recipient’s classic address.
  • Amount: For a Payment in XRP, the amount is specified in drops (1 XRP = 1,000,000 drops). This form accepts XRP and converts to drops.
  • Fee: XRP paid as the transaction fee (in drops). This XRP is destroyed (burned), not paid to validators.
  • Sequence: A per-account counter that orders transactions and prevents replay.
  • LastLedgerSequence: Optional. The last ledger index in which the transaction can be included; after that it becomes invalid.
  • Memos: Optional. Attach a note or reference (e.g. invoice ID) to the transaction. Stored on-ledger but not interpreted by the protocol.
  • Signature: Cryptographic proof that you authorize this transaction with the account’s private key.

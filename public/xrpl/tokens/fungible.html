Fungible Tokens on the XRP Ledger

Fungible Token Issuance Flow (Cold & Hot Wallet)

This walkthrough focuses on the cold / hot wallet pattern recommended when issuing fungible tokens on XRPL (Issue a Fungible Token). The theory for fungible tokens lives in the Tokens overview; here we focus on the practical steps.

Provide Issuer (Cold) and Hot Wallet Addresses

Long‑term issuing account. Configure settings such as Default Ripple, Require Auth, and transfer fees here.
Operational account that receives issued tokens from the cold wallet and forwards them to end users.

Issuance Steps

Each step below mirrors the issuing tutorial on xrpl.org. When an action completes successfully, the step is marked with a check based on the latest state of the involved accounts.

  • 1
    Check cold (issuer) account
    Verifies that the issuer address exists and is active on XRPL and has some XRP balance.
  • 2
    Check hot account
    Verifies that the hot address exists and is funded so it can create trust lines and send tokens.
  • 3
    Create trust line (hot → cold)
    In a real app, this would send a TrustSet transaction from the hot wallet to the cold address for the chosen currency code. Here, once you have created that transaction (for example using tools from the official tutorial), you can mark this step as done.
  • 4
    Send initial token payment (cold → hot)
    This corresponds to sending a Payment transaction from the cold issuer to the hot wallet for the configured amount and currency. After performing that transaction, use this button to record completion in the tutorial UI.

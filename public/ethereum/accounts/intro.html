Ethereum Accounts

What is an Ethereum Account?

On Ethereum, an account is an identity that can hold a balance (ETH and tokens), send transactions, and interact with smart contracts. There are two types: Externally Owned Accounts (EOAs) and Contract Accounts.

EOAs are controlled by a private key. The public key is derived from the private key using the same elliptic curve (secp256k1) you learned in the Public Key Cryptography section. The Ethereum address is derived from the public key (Keccak-256 hash, last 20 bytes).

Account Fields

Each Ethereum account is represented in the ledger state by four main fields:

  • Nonce: For an EOA, this is a counter of how many transactions have been sent from the account; it prevents replay attacks. For a contract account, it is the number of contracts that account has created.
  • Balance: The account’s balance in Wei, the smallest unit of ether (1 ETH = 10¹⁸ Wei).
  • Storage Root: A reference to the account’s storage trie, which holds all of a smart contract’s variables and their values. (EOAs have an empty storage root.)
  • Code Hash: The hash of the account’s code. For a contract, this is the hash of the deployed bytecode; the code is immutable. EOAs have an empty code hash.
Ethereum denominations: Wei, Gwei, and Ether

Ethereum uses multiple units for amounts of Ether:

  • Wei: The smallest unit; the base unit used by the protocol. 1 Wei = 1 Wei.
  • Gwei (Gigawei): Commonly used for gas prices. 1 Gwei = 10⁹ Wei = 0.000000001 ETH.
  • Ether (ETH): The standard unit. 1 ETH = 10¹⁸ Wei = 10⁹ Gwei.

Wei allows precise calculations; Gwei is convenient for gas; ETH is the human-readable unit.

Importing an Account

You can import an account by entering a private key (e.g. from the Key Generation section of Blockchain Concepts). The tool will derive and show the public key and Ethereum address.

Contract Accounts

Contract accounts are created when a smart contract is deployed. They have an address but no private key; their behavior is defined by their code. We cover them in the Smart Contracts section.

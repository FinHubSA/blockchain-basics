Introduction to Ethereum

What is Ethereum?

Ethereum is a decentralized, programmable blockchain that supports smart contracts and a native currency, Ether (ETH). It was designed to be a global platform for applications and value transfer without a central authority. Unlike simple payment chains, Ethereum allows developers to deploy code that runs on the network and governs assets and logic.

Key Concept: Ethereum uses Proof of Stake (since the Merge). Validators stake ETH and run nodes to propose and attest blocks. The network maintains a shared state (accounts, balances, contract code and storage) that updates when transactions are executed.

How It Works

On Ethereum:

  • Accounts: Externally owned accounts (EOAs) are controlled by private keys and can hold ETH and tokens and send transactions. Contract accounts are created by deployment and execute code when receiving transactions.
  • Transactions: Signed messages that transfer ETH, deploy contracts, or call contract functions. Users pay gas fees (base fee + priority fee) to have transactions included in blocks.
  • Consensus: Validators agree on the canonical chain and the resulting state. Block proposers earn block rewards and priority fees; the base fee is burned.

Key Properties

  1. Programmability: Smart contracts enable DeFi, NFTs, DAOs, and custom applications on a single chain.
  2. Ether (ETH): Native currency used for fees and as a store of value. Denominations include Wei (smallest), Gwei (for gas), and ETH.
  3. EVM: The Ethereum Virtual Machine runs contract bytecode in a deterministic way so every node reaches the same state.

Common Use Cases

  • Payments & transfers: Send ETH and tokens between accounts.
  • DeFi: Lending, borrowing, trading, and yield via smart contracts.
  • NFTs & tokenization: Unique and fungible tokens representing assets or membership.
  • DAOs: Decentralized governance and treasury management.
  • Identity & credentials: Verifiable credentials and attestations on-chain or tied to addresses.
